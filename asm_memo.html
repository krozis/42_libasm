Registres — anatomie
Un registre 64 bits se découpe en sous-parties accessibles séparément :
rax
rax (64b)
eax (32b)
ax (16b)
al
rdx
rdx (64b)
edx (32b)
dx (16b)
dl ← ton code
Même principe pour rbx→bl, rcx→cl, rdi→dil, rsi→sil…
Convention d'appel (System V AMD64)
rôleregistre
rdi1er argument
rsi2ème argument
rdx3ème argument
raxvaleur de retour
C'est le compilateur qui place les args dans ces registres avant d'appeler la fonction.

byte1 octet

dword4 octets

Nécessaire quand NASM ne peut pas déduire la taille seul, ex: cmp byte [rdi+rax], 0
Instructions clés
instructionce qu'elle fait
mov dst, src copie src dans dst — ne lit jamais le contenu de dst
test a, a ET bit-à-bit de a avec lui-même → met à jour les flags, ne stocke rien. Si a=0 : Zero Flag = 1
cmp a, b calcule a-b → met à jour les flags, ne stocke rien. Si a=b : Zero Flag = 1
je label "Jump if Equal" — saute si Zero Flag = 1
jmp label saut inconditionnel
inc reg incrémente reg de 1
ret retourne à l'appelant — rax est la valeur de retour
